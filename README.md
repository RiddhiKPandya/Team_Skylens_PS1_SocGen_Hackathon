## Project Structure

```text
Team_Skylens_PS1_SocGen_Hackathon/
│
├── 1. config/
│   ├── __init__.py
│   ├── config.py
│   └── __pycache__/
│
│   • Centralized configuration layer for dataset paths, model outputs,
│     dashboard integration, and pipeline settings.
│   • Ensures modular and scalable execution across all platform components.
│
├── 2. Datasets/
│
│   ├── 1. Raw_Original_datasets/
│   │   ├── identity_events.csv
│   │   └── identity_users.csv
│   │
│   │   • Original enterprise IAM datasets containing user identities,
│   │     privileges, systems access, and activity events.
│   │   • Acts as the source data for feature engineering and risk analytics.
│   │
│   ├── 2. Synthetic_datasets/
│   │   ├── identity_events_1800_combined.csv
│   │   └── identity_users_600_combined.csv
│   │
│   │   • Augmented dataset containing 600 identities and 1800 activity events.
│   │   • Includes 9 primary edge-case categories and 34+ sub-scenarios
│   │     covering identity sprawl, privilege abuse, orphaned accounts,
│   │     excessive access, anomalous behavior, and governance violations.
│   │
│   ├── 3. Feature_matrix/
│   │   └── feature_matrix.csv
│   │
│   │   • Enterprise feature repository containing 39 engineered IAM,
│   │     behavioral, privilege, activity, and governance features.
│   │   • Generates metrics such as Identity Sprawl Score, Blast Radius,
│   │     Sensitive Access Ratio, Service Account Risk, and After-Hours Activity.
│   │
│   ├── 4. Model_outputs/
│   │   ├── anomaly_output.csv
│   │   ├── final_risk_output.csv
│   │   └── rule_engine_output.csv
│   │
│   │   • Intermediate outputs generated by the Rule Engine,
│   │     Isolation Forest Anomaly Detector, and Risk Fusion Engine.
│   │   • Contains risk scores, anomaly severity, confidence metrics,
│   │     investigation priority, and risk classifications.
│   │
│   ├── 5. Final_Reports/
│   │   ├── final_soc_report.csv
│   │   └── findings_output.json
│   │
│   │   • Explainable SOC-ready reports containing findings,
│   │     evidence, root causes, and remediation recommendations.
│   │   • Serves as the bridge between analytics outputs and analyst workflows.
│   │
│   └── 6. LLM_reports/
│       └── USR00002.json
│
│       • AI-generated investigation narratives produced using
│         Ollama + Gemma 3 running locally.
│       • Converts technical detections into business impact analysis
│         and future investigation guidance.
│
├── 3. Src/
│
│   ├── 01. feature_engineering.py
│   │
│   │   • Generates 39 engineered features from identity and event datasets.
│   │   • Extracts behavioral, access, privilege, governance, and exposure
│   │     indicators used for downstream risk analytics.
│   │
│   ├── 02. rule_engine.py
│   │
│   │   • Implements 25+ deterministic IAM governance rules across
│   │     Identity Sprawl, Privilege Abuse, Stale Accounts,
│   │     Service Account Risk, Behavior Risk, and Data Access Risk.
│   │   • Produces explainable risk scores, confidence scores,
│   │     risk levels, and remediation actions.
│   │
│   ├── 03. anomaly_detection.py
│   │
│   │   • Uses Isolation Forest (200 Trees, 3% Contamination Rate)
│   │     to detect unknown and statistically abnormal identity behavior.
│   │   • Identifies high-risk users that bypass traditional rule-based systems.
│   │
│   ├── 04. risk_fusion.py
│   │
│   │   • Combines Rule-Based Risk (70%) and ML Anomaly Risk (30%)
│   │     into a unified enterprise risk score.
│   │   • Applies critical escalation logic and investigation prioritization.
│   │
│   ├── 05. explainability.py
│   │
│   │   • Generates transparent risk explanations from detected signals.
│   │   • Converts model outputs into analyst-readable findings and actions.
│   │
│   ├── 06. findings_generator.py
│   │
│   │   • Maps detections into structured IAM security findings.
│   │   • Produces JSON evidence for stale accounts, privilege abuse,
│   │     identity sprawl, sensitive access, anomaly detections,
│   │     and critical risk users.
│   │
│   └── 07. llm_analyst.py
│
│       • Integrates Ollama (Gemma 3) for fully offline AI-assisted analysis.
│       • Generates business impact assessments and investigation summaries
│         without external APIs, internet access, or third-party data exposure.
│
├── 4. Dashboard/
│
│   ├── app.py
│   │
│   │   • Streamlit-based SOC dashboard for enterprise identity risk monitoring.
│   │   • Provides visibility into users, systems, investigations,
│   │     anomalies, privilege exposure, and risk posture.
│   │
│   ├── identity_details.py
│   │
│   │   • Analyst investigation workspace with user-level drill-down analysis.
│   │   • Displays raw JSON findings, privilege graphs,
│   │     LLM interpretations, and investigation evidence.
│   │
│   └── requirements.txt
│
│       • Dependency manifest containing Pandas, NumPy, Scikit-Learn,
│         Streamlit, Plotly, Requests, and supporting libraries.
```
