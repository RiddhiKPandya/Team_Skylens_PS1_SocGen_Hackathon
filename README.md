## Project Structure

```text
Team_Skylens_PS1_SocGen_Hackathon/
│
├── 1. config/
│   ├── __init__.py
│   ├── config.py
│   └── __pycache__/
│
│   • Centralized configuration layer for managing paths, constants, and environment settings.
│   • Ensures consistent access to project resources across all modules.
│
├── 2. Datasets/
│
│   ├── 1. Raw_Original_datasets/
│   │   ├── identity_events.csv
│   │   └── identity_users.csv
│   │
│   │   • Original IAM user and activity datasets provided as source data.
│   │   • Serves as the foundation for feature engineering and risk analysis.
│   │
│   ├── 2. Synthetic_datasets/
│   │   ├── identity_events_1800_combined.csv
│   │   └── identity_users_600_combined.csv
│   │
│   │   • Expanded datasets containing engineered edge cases, attack scenarios,
│   │     and privilege abuse patterns.
│   │   • Improves model robustness by simulating real-world enterprise risks.
│   │
│   ├── 3. Feature_matrix/
│   │   └── feature_matrix.csv
│   │
│   │   • Consolidated feature repository containing 39+ IAM, behavioral,
│   │     privilege, and governance indicators.
│   │   • Used as the primary input for risk scoring and anomaly detection.
│   │
│   ├── 4. Model_outputs/
│   │   ├── anomaly_output.csv
│   │   ├── final_risk_output.csv
│   │   └── rule_engine_output.csv
│   │
│   │   • Stores outputs generated by the Rule Engine, Isolation Forest model,
│   │     and Risk Fusion Engine.
│   │   • Contains intermediate risk scores used for investigation prioritization.
│   │
│   ├── 5. Final_Reports/
│   │   ├── final_soc_report.csv
│   │   └── findings_output.json
│   │
│   │   • Human-readable SOC investigation reports and structured findings.
│   │   • Provides explainable evidence, risk reasoning, and remediation guidance.
│   │
│   └── 6. LLM_reports/
│       └── USR00002.json
│
│       • Analyst-ready interpretations generated using a local LLM.
│       • Converts technical findings into business impact and investigation insights.
│
├── 3. Src/
│
│   ├── 01. feature_engineering.py
│   │
│   │   • Generates behavioral, privilege, activity, and governance features
│   │     from raw IAM datasets.
│   │   • Builds the enterprise identity feature matrix used throughout the pipeline.
│   │
│   ├── 02. rule_engine.py
│   │
│   │   • Applies deterministic IAM governance rules to identify policy violations,
│   │     stale accounts, privilege abuse, and identity sprawl.
│   │   • Produces explainable rule-based risk scores and confidence metrics.
│   │
│   ├── 03. anomaly_detection.py
│   │
│   │   • Uses Isolation Forest to identify statistically abnormal identity behavior.
│   │   • Detects unknown risks beyond predefined governance rules.
│   │
│   ├── 04. risk_fusion.py
│   │
│   │   • Combines rule-based risk scores with ML anomaly scores.
│   │   • Produces a unified enterprise risk score and investigation priority.
│   │
│   ├── 05. explainability.py
│   │
│   │   • Generates human-readable explanations for detected risks.
│   │   • Maps technical findings to actionable remediation recommendations.
│   │
│   ├── 06. findings_generator.py
│   │
│   │   • Converts risk signals into structured IAM security findings.
│   │   • Creates evidence-driven JSON outputs for downstream analysis.
│   │
│   └── 07. llm_analyst.py
│
│       • Integrates Ollama (Gemma 3) for local AI-powered security analysis.
│       • Produces business impact assessments and investigation narratives
│         without external API dependency.
│
├── 4. Dashboard/
│
│   ├── app.py
│   │
│   │   • Main Streamlit dashboard for enterprise-wide risk monitoring.
│   │   • Provides visibility into users, systems, investigations, and risk posture.
│   │
│   ├── identity_details.py
│   │
│   │   • Detailed analyst workspace for user-level investigations.
│   │   • Displays JSON evidence, privilege graphs, and LLM interpretations.
│   │
│   └── requirements.txt
│
│       • Dependency manifest containing all required Python libraries.
│       • Enables reproducible deployment and execution.
```
